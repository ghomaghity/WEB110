Variable Typing

Typing refers to what type of data can be held inside of a variable. Variables in JavaScript are dynamically typed, meaning they can store whatever type can be assigned to them without issue. Below is an example of valid JavaScript code.

let t = 10
console.log(t) --Output: "10"
t = "a string!"
console.log(t) --output: "a string!"

Java is statically typed. This means that whatever type a varaible is designated to hold when declared is the only type it will ever be able to hold, at least without casting it to a different type.

int t = 10
System.out.println(t) --Output: "10"
t = "a string!"--Type mismatch: cannot convert from String to int


Compilation Methods

Java is just-in-time (JIT) compiled into a universal bytecode. This bytecode can then be run on multiple systems using runtime environments specifically made for those systems. This allows programmers to write code on one system and be fairly confident it will work on another without having to maintain multiple codebases.

JavaScript started off as an interpreted language, meaning programs were run through line by line. This also allows developers to write system-independent software, at the tradeoff of code generally being slower. Relatively recently, however, JavaScript has gotten multiple means of JIT compilation.
